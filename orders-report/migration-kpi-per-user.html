Migration: KPI per-user attribution

Re-tính KPI cho mọi đơn theo người thực sự upsell trong audit log (thay vì chủ đơn theo STT).

⚠️ BẮT BUỘC backup PostgreSQL trước khi chạy: 
pg_dump $DB_URL -t kpi_statistics > kpi_statistics_backup_$(date +%Y%m%d).sql
Logic:
  1. Fetch toàn bộ orderCode từ kpi_base
  2. Với mỗi orderCode → DELETE cũ + PATCH lại theo per-user audit log
  3. Throttle 100ms giữa các request
  4. Đơn không có upsell → bị xóa khỏi tab KPI
Tổng đơn: — Đã xử lý: 0 Lỗi: 0 Bỏ qua (no audit): 0
