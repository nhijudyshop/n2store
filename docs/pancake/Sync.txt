Sync
The Sync feature helps you share and manage common content across multiple pages, saving time and keeping everything consistent.
I. Overview
The Sync feature allows you to share the same data across multiple pages in Pancake. This means you only need to update once, and all pages in the sync group will automatically have the same content.
There are 3 types of synchronization:
	•	🖼️ Media & File
	•	💬 Quick reply
	•	🏷️ Conversation Tag

II. 🖼️ Media & File sync
1. Purpose
Allows pages in the same group to share a common Media & File library. When you add or edit a Media/ File on one page, the other pages in the group will automatically update.
Example: You have 3 pages selling the same product. When you upload a new product Media/ File to Page A, Pages B and C will also have that Media/ File in their library.
2. How to do it
✅ Step 1: Go to Setting → Sync → Media & File.
https://statics.pancake.vn/pancake-docs/2025/10/14/86b941b9-905d-4922-a386-12d3cf2575dd_iScreen Shoter - Google Chrome - 251014221901.jpg
✅ Step 2: Select Create.
https://statics.pancake.vn/pancake-docs/2025/10/14/2e37686d-79d8-436e-a6f7-d48e0305ec30_iScreen Shoter - Google Chrome - 251014221951.jpg
✅ Step 3: Name the group, for example: “Shared Product Images.”
✅ Step 4: Select at least 2 pages you want to sync Media/ File between.
✅ Step 5: Click Sync to complete.
Once the group is created, any new Media/ File added to one page in the group will automatically appear on the other pages.
📌 Note: 
- You need to select at least 2 pages to sync.
- Each page can only belong to one sync group. (For example, in the image above, the page status will show: “This page already belongs to another sync group").
- You must be an Admin of the page to enable syncing. 
- Once a page joins a sync group, all new images uploaded will be stored in the group’s shared library. If the page leaves the group, it will no longer have access to that shared library.
III. 💬 Quick Reply Sync
1. Purpose
Allows pages in the same group to share a common set of quick replies. This ensures that staff across different pages use a consistent set of response templates.
Example: You want all staff on your pages to have ready-to-use templates like:
“Yes, the product is still available.”
“Please leave your phone number so our team can contact you for consultation.”
By creating a sync group, all pages in that group will automatically share the same quick reply templates.
2. How to do it
✅ Step 1: Go to Setting → Sync → Quick reply

✅ Step 2:  Select Create.
https://statics.pancake.vn/pancake-docs/2025/10/14/dbfc7b67-e3af-445f-98bc-c7cc8cc2d34a_iScreen Shoter - Google Chrome - 251014224234.jpg
✅ Step 3: Name the group, for example: “Customer Support Replies.”.
✅ Step 4: Select the Media/ File Sync group.
✅ Step 5: Choose the pages you want to include in the Quick Reply Sync group.
✅ Step 6: At this step, you’ll see the option “Merge all data from the selected pages into a new shared library.” If you:
	•	Don’t select it: The system will only sync new data added after the page joins the group.
	•	Select it (tick the box): The system will sync all data from the page — including both old and new data — into the new shared library.
➡️ After making your choice, click Sync to complete the process.
From then on, any new Quick Replies created on one page in the group will automatically appear on all other pages in the group.
📌 Notes: 
- The synced pages must belong to the same group as the previously created image sync group. → If you haven’t created an image group yet, you’ll need to create one first. - You need to select at least 2 pages to sync. 
- You must be an Admin of the page to enable syncing. 
- Each page can only belong to one Quick Reply sync group.
3. How It Works and What Happens When Leaving a Group
Merging data when creating a group:
	•	All existing Quick Replies from the selected pages will be combined into a shared library.
	•	If there are any duplicates, the system will keep the latest version (based on the Quick Reply ID) and avoid creating duplicates.
When a page leaves the group:
	•	That page will no longer have access to the shared library.
	•	However, any Quick Replies that already existed on the page before leaving will remain available on that page.
	•	New Quick Replies added to the shared library after the page leaves will not appear on that page.
Example:
	•	Page A has 50 Quick Replies, and Page B has 70 Quick Replies.
	•	When creating the sync group → the shared library will contain 120 Quick Replies (if there are no duplicate IDs).
	•	When Page A leaves the group → it keeps its original 50 Quick Replies and loses access to the shared library.
	•	Meanwhile, Page B and other pages still share and use the library with 120 Quick Replies.
IV. 🏷️ Conversation Tag Sync
1. Purpose
Helps you standardize conversation tags across all pages, ensuring that customer reports and support processes remain consistent.
Example: 
All pages share the same tags, such as: “Potential Customer”, “Purchased”, “Needs Follow-up”
When you add or edit a tag on one page, the other pages in the same group will automatically update.
2. How to use it
✅ Step 1: Go to Setting → Sync → Conversation Tag

⚙️ How Conversation Tags Work When Synced Across Pages
Before syncing from Page A to Page B, it’s important to understand how Conversation Tags function:
1. How It Works
When you add pages to a sync group, the system will combine all existing tags from those pages into one shared tag library for the entire group.
All pages in the group will then use the same shared tag set, and their old, individual tag sets will no longer be used.
2. Example
Page A has tags: 1, 2, 3
Page B has tags: 3, 4, 5
➡️ After syncing, the system will create a new shared tag set: G1, G2, G3, G4, G5.
At that point, both Page A and Page B will use this new shared tag set and will no longer have their old tags.
📌 Notes: 
- Since tag “3” from Page A and Page B are identical in content, the system will automatically merge them into a single tag in the new tag set.
- The new tag set is a combined copy of all tags from the member pages. 
- The old tags will be permanently deleted and cannot be restored. 
- After leaving the group, the pages will continue using the synced tag set and cannot revert to their old tags.
3. Update time:
The system may take a few minutes to several hours to fully sync the new tags across all conversations, depending on the number of conversations each page has. During this time:
	•	The tag list will already show the new shared tags, and you can still tag and use features as usual.
	•	Some conversations may not update immediately, so tags or filters might not display completely until the sync is finished.
✅ Step 6: After reviewing the notice, if you agree with the conditions, select Agree to proceed.
📌 Notes: 
- You must select at least 2 pages to sync. 
- You must be an Admin of the page to perform syncing. 
- Each page can only belong to one Conversation Tag sync group. 
- Once a page joins the group, its tags will be merged into the group’s shared tag library. If the page leaves the group, it will no longer see or edit the tags in the shared library.
3. How It Works and What Happens When Leaving a Group
Merging data when creating a group:
	•	All existing tags from the selected pages will be combined into a shared library.
	•	If there are duplicate tags across pages, the system will keep only one tag (comparison is based on the tag name only, not the color) and avoid creating duplicates.
When leaving a group:
	•	The page will no longer have access to the shared library.
	•	However, the page will retain all of its own tags, including both the original tags and any tags added during the sync process.
Example:
	•	Page A has 50 conversation tags, and Page B has 70 conversation tags.
	•	When creating the sync group, all tags are merged into a shared library:
When leaving the group:
	•	Page A leaves the group → it loses access to the shared library.
	•	Page A still keeps all tags it has (including both its original tags and those synced during the process — 120 conversation tags).
	•	Page B and other pages in the group continue using the shared library (120 conversation tags).

VII. Conclusion
The Sync feature helps you manage multiple pages more efficiently, keep content consistent, and avoid repeating the same actions across pages.
