Luồng dữ liệu chính

Web 2.0 được chia thành 3 pipeline song song, mỗi pipeline có entry point rõ ràng và output đẩy ngược về master data.

01

Bán hàng (Shop → Khách hàng)

  1. FB Comment / Web — tpos-pancake nhận comment livestream
  2. native-orders — auto-extract đơn từ comment
  3. fast-sale-orders (PBH) — confirm tạo phiếu bán hàng
  4. reconcile — quét barcode, đóng gói, bàn giao shipper
  5. customer-wallet — refund cộng lại ví KH nếu có
02

Mua hàng (Shop ← NCC)

  1. so-order — tạo đơn mua NCC theo tab quốc gia
  2. web2-products (pending stock) — đẩy SL về pending
  3. confirm-purchase — hàng về cộng stock thực
  4. purchase-refund — phiếu trả hàng giảm stock
  5. supplier-debt / supplier-wallet — tính công nợ NCC
03

Tiền (SePay → Ví)

  1. SePay webhook — bank ghi nhận thu/chi
  2. web2_balance_history — log giao dịch (bảng Web 2.0 độc lập, dual-write)
  3. match customer / NCC — auto hoặc thủ công (link)
  4. customer-wallet / supplier-wallet — credit vào ví tương ứng
  5. accountant — duyệt, gửi reminder, audit adjustments

Cơ sở dữ liệu & Router Web 2.0

Web 2.0 đã tách hoàn toàn khỏi Web 1.0: Postgres riêng n2store-web2-db, API namespace riêng /api/web2/*, và Firebase collection prefix web2_. ✅ Cutover xong 2026-06-03 — 26 route web2 + webhook + crons đọc/ghi web2Db; Web 1.0 (n2store_chat) KHÔNG bị đụng.

🟢 Postgres Web 2.0 — n2store-web2-db

Render PG riêng (Singapore, dpg-d8d7be…, db n2store_web2) — pool web2Db ← WEB2_DATABASE_URL. Đích cuối của mọi bảng Web 2.0.

  • web2_customers — kho KH riêng (sync TPOS) ✅
  • web2_records · web2_entities — generic store ✅
  • ✅ Đã ở web2Db (cutover xong): web2_products · web2_variants · web2_balance_history · web2_customer_wallets · web2_wallet_transactions · web2_pending_matches · web2_payment_qr_codes · web2_match_audit · web2_notifications · web2_users · web2_kpi_* · native_orders · fast_sale_orders

⚪ Postgres Web 1.0 — n2store-chat-db

Render PG (dpg-d4kr80…, db n2store_chat) — pool chatDb ← DATABASE_URL. Chứa data Web 1.0; bảng web2_* tạm còn ở đây trong lúc migrate.

  • Web 1.0: customers · customer_wallets · balance_history · app_users …
  • Web 2.0 KHÔNG còn gọi: /api/v2/customers → /api/web2/customers (kho riêng).

🔥 Firestore — prefix web2_

  • web2_so_order/main — Sổ Order (local-first sync)
  • web2_supplier_wallet/main — Ví NCC
  • web2_suppliers/main — danh sách NCC
  • web2_customer_wallet/main — UI mirror ví KH

Realtime: KHÔNG dùng Firestore listener — dùng SSE pub/sub trên Render, hub riêng Web 2.0 (/api/realtime/web2/sse, topic web2:<entity>).

📡 Realtime SSE — phủ sóng tất cả trang & cách dùng (ĐỌC KHI CODE TRANG MỚI)

Web 2.0 dùng 1 server SSE chung trên Render (render.com/routes/realtime-sse-web2.js, endpoint /api/realtime/web2/sse) cho mọi cập nhật UI realtime — KHÔNG Firestore listener, KHÔNG polling. Mỗi mutation backend gọi _notify('web2:<entity>', …) sau khi COMMIT; client Web2SSE.subscribe('web2:<entity>', cb) + debounce → reload. Debug live tại SSE Monitor (Admin). Chi tiết: docs/web2/SSE-REALTIME.md.

✅ Bảng phủ sóng realtime — toàn bộ trang trên menu (audit 2026-06-04)

Tất cả trang đều ĐÃ triển khai (REAL) — không trang nào là stub. Cột Realtime: ✅ SSE = realtime qua SSE chuẩn; 🔸 legacy = realtime qua Firestore/WebSocket cũ (nên migrate SSE); ⚪ N/A = không cần realtime (báo cáo tính toán / cấu hình / export 1 lần / on-demand).

🗄️ Database & Pool

  • 2 Postgres tách biệt: n2store-web2-db (Web 2.0, pool web2Db ← env WEB2_DATABASE_URL) vs n2store-chat-db (Web 1.0, pool chatDb ← DATABASE_URL).
  • Route/service Web 2.0 LUÔN lấy pool: const db = req.app.locals.web2Db || req.app.locals.chatDb; (web2Db chính, chatDb fallback khi env unset/dev).
  • TUYỆT ĐỐI KHÔNG ghi (INSERT/UPDATE/DELETE) vào bảng Web 1.0 từ code Web 2.0. Bảng dùng chung tên (vd customers) → web2Db giữ bản copy riêng.
  • Firestore: collection Web 2.0 prefix web2_<entity>/main (vd web2_so_order, web2_supplier_wallet).

🏷️ Đặt tên

  • Bảng Postgres: prefix web2_<entity> (vd web2_products, web2_balance_history).
  • Router/API: /api/web2/<entity> (mount dưới /api/web2/*) hoặc /api/web2-<entity>. KHÔNG dùng /api/v2/* (namespace cũ — Web 1.0/mixed).
  • Service file: web2-<name>.js; hàm nhận db (pool) làm tham số đầu, KHÔNG hardcode pool.
  • Hàm: camelCase, prefix domain rõ (vd insertWeb2BalanceHistory, processWeb2Match, upsertWeb2Customer).
  • File mới: thêm marker | WEB2.0 module. vào header #Note (HTML/JS dòng 1).

⚡ Realtime & Patterns

  • Realtime: KHÔNG Firestore listener. Dùng SSE hub Web 2.0 /api/realtime/web2/sse, topic web2:<entity>. Server gọi web2RealtimeSseRoutes.notifyClients(topic, data) sau commit; client Web2SSE.subscribe('web2:<entity>', cb).
  • Mutation UI-first: dùng Web2Optimistic.run({...}) (UI sync ngay, backend nền, rollback nếu lỗi). Ngoại lệ giữ await: thao tác tiền.
  • Modal: class modal-content + modal-body (auto anti-lag). KHÔNG backdrop-filter: blur().
  • Sidebar/shell: dùng web2/shared/tpos-sidebar.js + web2-tpos-theme.css. KHÔNG import code từ Web 1.0 (orders-report, inbox…).

📏 Luật trích xuất

  • Mọi dãy 5–10 chữ số trong content đều lấy làm phone-tail candidate để tìm KH (đuôi SĐT).

  • Lọc blacklist (STK shop 75918, SĐT test) + isObviousTestPhone().

🎯 Matcher quyết định

  • Mỗi candidate → search TPOS Partner (endsWith đuôi) → gom unique phone (dedup).
  • 1 unique phone → auto credit ví ( exact_phone/single_match).

  • 0 match → giữ "Chưa gán", badge hiện đuôi để gán tay.
