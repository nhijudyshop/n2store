Máy in

Quản lý nhiều máy in & gán máy cho từng chức năng (in PBH ra máy bill, in mã SP ra máy tem…). Lưu theo từng máy POS.

⚠️ Print Bridge chưa chạy (hoặc bị trình duyệt chặn). Mở terminal trên máy POS chạy: node scripts/print-bridge.js (giữ cửa sổ mở). Nếu web chạy bản HTTPS mà vẫn báo chưa chạy → khởi động lại bridge bản mới (đã thêm hỗ trợ Private Network Access). "Test kết nối" lỗi EHOSTUNREACH = bridge OK nhưng máy in chưa bật / khác mạng / sai IP.

Cài máy POS (Windows — 1 click): Print Bridge + Giọng cao cấp

1 file .bat cài cả 2: Print Bridge (in thẳng máy in IP) + Giọng cao cấp (clone giọng cho video). Bấm đúp 1 lần: tự cài, chạy nền, tự bật mỗi khi mở máy và tự tắt các bản auto cũ. Lần đầu tải model giọng ~595MB (vài phút) — máy sẽ tự hiện ở trang Tạo video.

  1. Bấm Tải file cài đặt (.bat) → lưu về máy POS.
  2. Bấm đúp file cai-may-pos.bat (nếu Windows hỏi SmartScreen → "More info" → "Run anyway"). Lần đầu chờ tải model giọng ~595MB.
  3. Xong! Cả Print Bridge + Giọng cao cấp chạy nền + tự bật khi khởi động. Máy tự hiện ở trang Tạo video (mục Giọng cao cấp) + bấm Test kết nối ở máy in.
  4. Gỡ: xóa file %APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\N2StorePrintBridge.vbs
Cài máy chấm công DG-600 (Windows — 1 click)

1 file .bat tự tải agent chấm công về máy rồi cài luôn: nhận dữ liệu máy DG-600 (vân tay) đẩy lên trang Chấm công. Bấm đúp 1 lần: tự tải, tự gỡ bản cũ, tự kiểm tra kết nối, chạy nền + tự bật khi mở máy.

  1. Bấm Tải file cài Chấm công (.bat) → chép sang máy tính trong shop (cùng mạng LAN với máy DG-600).
  2. Bấm đúp cai-cham-cong.bat (nếu SmartScreen → "More info" → "Run anyway"). Cần Node.js — https://nodejs.org.
  3. Đọc dòng [8/8]: lấy IP máy tính → vào máy DG-600 (menu Comm/Cloud/ADMS) đặt Server address = IP đó, Port 8081, Mode Auto upload.
  4. Chấm 1 vân tay → mở trang Chấm công (Quản trị viên) thấy "Đang kết nối" + lượt chấm hiện ra.
  5. Gỡ: bấm Tải file gỡ → bấm đúp file go-cham-cong.bat.
